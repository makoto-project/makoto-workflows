# Documentation

Three ways to run the DBOM pipeline — pick the one that fits your workflow.

| Approach | How | Best For |
|----------|-----|----------|
| [**Local CLI**](local-cli.md) | `just demo` | Development, testing, quick iteration |
| [**Actions Workflow**](actions-workflow.md) | Push or manual dispatch | CI/CD, deterministic gating |
| [**Agentic Workflow**](agentic-workflow.md) | Push, dispatch, or schedule | AI-assisted auditing, smart reporting |

All three use the same [makoto-project/makoto-cli](https://github.com/makoto-project/makoto-cli) CLI toolkit under the hood.
